# 三维评分标准

## 概述

SPEC V4 使用三个正交维度对每张配图进行 0-100 评分，驱动场景设计的投入深度。

## 维度定义

### 1. 结构骨架（Structure）

**核心问题**：这张图需要展示信息之间的逻辑关系吗？

| 分数段 | 含义 | 典型场景 |
| --- | --- | --- |
| 80-100 | 核心是信息关系，必须精确展示 | 流程图、对比图、层级图、矩阵 |
| 50-79 | 有结构但不是重点 | 有简单排列但主要靠隐喻传达 |
| 30-49 | 结构是背景 | 简单布局即可 |
| 0-29 | 无结构需求 | 纯情绪或纯装饰 |

**设计输出（≥70）**：
- 信息逻辑关系类型（并列、递进、对比、包含）
- 空间排布方案
- 面板数量和排列方式

### 2. 隐喻包装（Metaphor）

**核心问题**：这张图需要将抽象概念转化为视觉意象吗？

| 分数段 | 含义 | 典型场景 |
| --- | --- | --- |
| 80-100 | 核心是隐喻，必须创造强意象 | 锚定抽象概念、原文纯理论 |
| 50-79 | 需要隐喻辅助理解 | 概念有些抽象但原文有暗示 |
| 30-49 | 有暗示就融入 | 概念本身较具象 |
| 0-29 | 不需要隐喻 | 信息关系已经足够清晰 |

**设计输出（≥70）**：
- 触发隐喻 brainstorm
- 原文有具象意象 → 直接采纳
- 原文无具象意象 → 发散 3 个候选 → 三维评分 → 择优

**隐喻候选评分标准**：

| 维度 | 含义 | 评分要点 |
| --- | --- | --- |
| **semantic_fit**（语义契合度） | 读者能从画面联想到原文概念吗？ | 直觉关联强=高分，需要解释=低分 |
| **ai_renderable**（AI 可渲染性） | 全是名词？物件清晰？AI 画得好？ | 常见具象物件=高分，抽象/动态=低分 |
| **style_compatible**（风格兼容性） | 和当前风格指南的视觉语言匹配吗？ | 几何/理性=高分，卡通/写实=低分 |

**total** = semantic_fit + ai_renderable + style_compatible

**选择策略**：
- 快速模式：自动选 total 最高者
- 交互模式：展示 3 个候选让用户选择

### 3. 情绪修辞（Emotion）

**核心问题**：这张图需要传递特定的情绪感受吗？

| 分数段 | 含义 | 典型场景 |
| --- | --- | --- |
| 80-100 | 核心是情绪，必须精准渲染 | 渲染感受、制造冲击、唤起共情 |
| 50-79 | 需要情绪辅助 | 内容有感受倾向但不是主体 |
| 30-49 | 有暗示就融入 | 轻微情绪色彩 |
| 0-29 | 不需要情绪 | 纯信息展示 |

**设计输出（≥70）**：
- 目标情绪定义
- 情绪强度：轻触 / 中等 / 浓烈
- 视觉手法：色温偏移、光影对比、材质质感、空间压缩/扩张

## 默认权重表

以功能层为起点，根据原文内容上下调整：

|  | 结构骨架 | 隐喻包装 | 情绪修辞 |
| --- | --- | --- | --- |
| **节奏层** | 0 | 0 | 20 |
| **情绪层** | 20 | 50 | 80 |
| **锚定层** | 20 | 80 | 50 |
| **结构层** | 80 | 20 | 20 |

## 评分规则

1. 默认权重是起点，不是终点
2. 根据原文具体内容上下调整（可能偏离默认值 ±30）
3. 每个评分附一句话依据
4. 评分完成后直接进入 Step 2，不需要确认

## 隐喻类型参考

- **容器** — 盒子、抽屉、房间、大脑
- **路径** — 道路、河流、轨道、阶梯
- **工具** — 望远镜、钥匙、滤网、杠杆
- **生态** — 种子→树、土壤、天气、季节
- **空间** — 地图、坐标系、楼层、地形

**选择优先级**：AI 渲染稳定性 > 信息传达力 > 艺术表现力
