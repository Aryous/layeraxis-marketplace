# Output Format & Constraints (输出规范与边界约束)

为每一幅设计的图**单独创建一个文件**，保存在 `imgs-spec/` 目录中。

**文件命名规范**：`imgs-spec/NN-[slug].md` (例如 `imgs-spec/01-metaphor-gap.md`)

每份文件按照以下多段式模板编写：

## 模板

```markdown
# 创意设计卷宗：[概念焦点]

### Step 1：理解与拆解 (深耕原文)+定位与评分
- **核心概念**：[提炼1句话总结原文核心议题]
- **主要目的**：[为了解释流程/横向对比/还是揭示原理？]
- **叙事骨架**：[并列 / 对比 / 递进 / 因果]
- **关键实体字典**：[提取原文中自带的所有具象名词，不可遗漏]
- **功能层**：[锚定层/结构层/情绪层/节奏层]
- **三维评分**：结构骨架 XX / 隐喻包装 XX / 情绪修辞 XX
> 评分依据：[一句话原因]

### Step 2：设计思考 (先发散后收敛)
- **排版考量**：[解释为什么选此排版？容器是什么？]
- **隐喻推演**：[原文是否有具象意象？发散候选隐喻，说明为什么选定最终的物理物件。]
- **情绪渲染**：[如需要，描述色温、光影等情绪手法，否则跳过]

### Step 3：中文场景定稿
全局：[构图方向、比例、整体布局]
容器：[卡片或分区的具体物理规格]
逐面板/分区：[严密的方位词 + 具体物件描述，颜色就地标注 Hex 值]
连接元素：[虚线、总结文字等]

---

## English Prompt

按分层骨架逐段翻译，每层之间空一行：
```

```md
第一段：全局声明
  风格关键词 + 构图方向和比例（如 wide horizontal composition 16:9）

第二段：容器规格（多面板图必须有）
  面板/卡片的物理定义：数量、排列、背景色、描边、圆角、阴影

第三段起：逐面板场景描述
  用 --- Card N --- 分隔
  每个面板内：物件描述 + 颜色 inline 声明（如 thin amber border #f59e0b）

倒数第二段：连接元素
  面板间的连接线、底部文字

最后一段：风格基底 + 语言声明
  从风格指南加载的通用风格描述
  All visible text in Chinese. No English text.
```
## 负面提示词
从风格指南加载

**颜色 inline 规则**：每个物件在首次出现时就地标注颜色 Hex 值，不在末尾集中声明。
