## 输入
- density{（standard（5～8 张）/full（10～15 张）/minimal（3～6 张）}
- style-guide{风格指南}
- 文章正文
- 用户附加偏好（可选）

## 配图哲学
> **先看内容，再想画面。** 内容的自然结构是最好的构图线索。三个并列问题就画三张卡片，一个因果链就画上下两层。
> **原文意象优先。** 原文有具象素材就直接用，不要跳过原文去造隐喻。
> **AI 画名词，不画动词。** 把「时间流逝」翻译成「融化的沙漏」，把动词变成可画的东西。
> **设计和翻译分离。** 中文想清楚画面，英文只做翻译，不在翻译时做设计决策。
> **评分驱动深度。** 维度评分决定每个维度投入多少认知资源，不多做也不少做。

## 核心架构

```jsx
功能层定位 → 三维动态评分(0-100) → 场景设计(中文) → 风格+上色 → 翻译(英文) → 迭代
```

- **四层功能模型**（节奏/情绪/锚定/结构）决定评分起点
- **三个正交维度**（结构骨架 × 隐喻包装 × 情绪修辞）的投入深度由评分动态决定
- **风格基底**独立于内容生成，通过加载风格指南获取

## 执行流程

### Step 1：阅读分析与大纲规划

阅读用户提供的全文文字，完成以下规划并先行输出：

**1. 理解文章脉络**
- **文章意图**：整体是在解释流程、对比优劣、还是布道理念？
- **叙事逻辑**：段落之间是并列、对比、递进还是因果关系？
- **提炼实体**：圈出所有原文提及的可视化具象词（沙漏、积木、齿轮）。

**2. 拆图规划与大纲输出**
- 根据配图 `density` 参数规划图片数量和拆分节奏。
- **输出第一份文件**：`imgs-spec/outline.md`（格式见 @references/outline-format.md）
- 这个大纲只做宏观规划（包含有几张图、每张图的概念焦点和主要出图目的），不展开详细设计。

### Step 2：单图深度推演 (NN-slug.md)

> 文档即思考：你为每张图创建的 `NN-slug.md` 不是设计完成后的报告，写文档的过程本身就是设计过程。边写边思考，写完即定稿。参考输出规范@references/output-format.md

大纲划定后，为每一幅图**分别创建独立的 `NN-slug.md`**，在文件中依次完成：
**1. 理解内容**

- **核心概念**：这段话在讲什么？
- **叙事结构**：概念之间的关系（对比、递进、并列、因果）
- **关键实体**：涉及哪些具体对象

**2. 功能层定位**
| 功能层 | 主要目的 | 识别信号 |
| --- | --- | --- |
| **节奏层** | 调节阅读节奏，给眼睛一个呼吸点 | 段落间需要视觉休息，不承载具体信息 |
| **情绪层** | 为接下来的文字预设情绪基调 | 文字在渲染感受、制造冲击、唤起共情 |
| **锚定层** | 用具象物件锚定抽象概念 | 文字在解释抽象概念，需要「让人一看就懂」的载体 |
| **结构层** | 展示信息的逻辑关系 | 文字在描述流程、对比、层级等关系 |

**3. 三维动态评分（0-100）**
以功能层默认权重为起点：
|  | 结构骨架 | 隐喻包装 | 情绪修辞 |
| --- | --- | --- | --- |
| **节奏层** | 0 | 0 | 20 |
| **情绪层** | 20 | 50 | 80 |
| **锚定层** | 20 | 80 | 50 |
| **结构层** | 80 | 20 | 20 |

- **≥ 70**：完整设计输出，在此维度深度发散
- **30-69**：简要考虑，轻度融入
- **< 30**：跳过，不投入多余认知资源

**4. 场景设计与最终翻译**

对每张图，按评分高低决定每个维度的投入深度。**所有设计思考用中文完成，本步不写英文。**

#### 评分 ≥ 70 的维度：完整设计

**结构骨架 ≥ 70 时：**

- 信息之间是什么逻辑关系？（并列、递进、对比、包含）
- 用什么空间排布最直觉？
- 确定面板数量和排列方式

**隐喻包装 ≥ 70 时，触发隐喻 brainstorm：**

<aside>
**原文有具象意象** → 直接采纳，检验三项标准即可
**原文无具象意象** → 发散 3 个候选隐喻 → 择优 → 采纳
</aside>

隐喻 brainstorm 流程：

  1. 针对目标概念，生成 **3 个候选隐喻意象**
  2. 每个用一句话描述：「什么物件 + 什么状态 + 传递什么感受」
  3. 候选之间尽量走不同隐喻类型（容器/路径/工具/生态/空间）
  4. 按三项标准快速择优：
      - **语义契合度** — 读者能从画面联想到原文概念吗？
      - **AI 可渲染性** — 全是名词？物件清晰？AI 画得好的常见物件？
      - **风格兼容性** — 和当前风格指南的视觉语言匹配吗？
  5. 选最优进入场景设计

**情绪修辞 ≥ 70 时：**

- 这张图要传递什么感受？
- 情绪强度：轻触 / 中等 / 浓烈
- 视觉手法：色温偏移、光影对比、材质质感、空间压缩/扩张

#### 评分 30-69 的维度：简要融入

如果原文暗示了相关元素，一两句带过融入场景，不独立展开。

#### 评分 < 30 的维度：跳过

不投入认知资源。

#### 场景组装

把所有维度的设计合并为完整的中文场景描述，按**分层骨架结构**输出：

```
全局：构图方向、比例、整体布局
容器：多面板时的卡片/面板物理规格（尺寸、描边、圆角、阴影）
逐面板：每个面板的物件 + 空间关系（前后/上下/内外）
连接元素：面板间的虚线、箭头、底部文字
情绪基调：整体的情绪色彩（如有）
```

#### 场景检验（两项，快速过）

- **名词检查**：每个元素都是 AI 画得好的具象名词吗？动词必须替换为物件
- **3 秒测试**：读者扫一眼能看懂画面在表达什么吗？看不懂就简化

### Step 3：加载风格 + 上色

**1. 确定风格指南**
- 加载要求的风格指南 `@references/styles/{style_name}.md`
**2. 从风格指南提取**

- 风格基底描述（通用视觉描述段）
- 色彩系统（核心配色 Hex 值）
- 负面提示词

**3. 给场景上色**

- 给 Step 2 中文场景里的每个元素分配具体颜色 Hex 值
- 颜色跟物件走，不集中声明
- 强调色用于关键/异常元素，辅助色用于背景和框架
- 节奏层只需要基础色调，不需要精细分配


### Step 4：写提示词（翻译）

**输入** = Step 2 中文场景 + Step 3 颜色分配

**纯翻译，零设计决策。** 不在这一步新增物件、改变空间关系或调整颜色。

**1. 图片概念卡（中文，简短）**

```
概念焦点：一句话说明这张图要表达什么
功能层：X
维度评分：结构 XX / 隐喻 XX / 情绪 XX
视觉核心：一句话描述画面的核心创意
```

**2. 英文提示词**

按分层骨架逐段翻译，每层之间空一行：

```jsx
第一段：全局声明
  风格关键词 + 构图方向和比例（如 wide horizontal composition 16:9）

第二段：容器规格（多面板图必须有）
  面板/卡片的物理定义：数量、排列、背景色、描边、圆角、阴影

第三段起：逐面板场景描述
  用 --- Card N --- 分隔
  每个面板内：物件描述 + 颜色 inline 声明（如 thin amber border #f59e0b）

倒数第二段：连接元素
  面板间的连接线、底部文字

最后一段：风格基底 + 语言声明
  从风格指南加载的通用风格描述
  All visible text in Chinese. No English text.
```

**颜色 inline 规则**：每个物件在首次出现时就地标注颜色 Hex 值，不在末尾集中声明。

**3. 负面提示词**（从风格指南加载）

### Step 5：迭代修正

当用户贴回生成的图片或表示不满意时：

1. **对比**：提示词与实际效果的差异
2. **归因**：
    - 场景描述不够具体？→ 回 Step 2 补充细节
    - 隐喻不对？→ 回 Step 2 从 brainstorm 的其他候选中挑选，或重新 brainstorm
    - 颜色/风格不匹配？→ 回 Step 3 调整
    - AI 模型理解偏差？→ 在 Step 4 调整措辞
3. **按需加载参考页**：如果连续两次修正仍不满意，加载最佳实践参考页对照校准
4. **输出修正版提示词**
5. **沉淀**：如发现风格指南缺失的规范，建议更新到风格指南页面

## 硬规则

- **标注数量**：每张图标注 ≤ 2 处，标注是辅助不是主体
- **标注逐字嵌入**：中文标注在英文提示词中逐字写出，格式 `Chinese text "具体文字" at [位置]`，禁止 `Chinese title label` 等模糊指令
- **语言声明**：每条提示词末尾 `All visible text in Chinese. No English text.`
- **禁止 emoji**：提示词中不得出现 emoji，用纯英文描述替代
- **元素上限**：可辨识视觉单元 ≤ 7，超过就砍，优先留白
- **面板物件预算**：每个面板/卡片内核心物件 ≤ 3 个，动作暗示 ≤ 1 个
- **视觉域一致性**：多面板图所有面板在同一物件语言域内取材，禁止混搭
- **提示词语言**：默认英文提示词，图内标注默认中文
- **不做的事**：不生成图片本身，只输出提示词；不做 UI 设计稿

## 多元素排版规范

当一张图需要展示多个平行信息时，根据内容特点从以下两种排版策略中选择一种。不要无脑默认卡片。

### 策略 A：卡片画廊 (Card Gallery)

**适用场景**：独立并列的模块展示（如三个问题、四个功能），信息块之间没有动作关联或对比张力。

- **画布**：默认 16:9 横向宽幅，竖向用 9:16
- **容器**：每个信息块用物理卡片承载（白色背景、描边、圆角 12、双层阴影）
- **视觉导航**：卡片左上角加编号（"1" / "2" / "3"），深色粗体，编号下方加中文副标题
- **连接**：底部水平虚线连接所有卡片，虚线下方放总结性文字
- **面板内约束**：核心物件 ≤ 3 个，动作暗示 ≤ 1 个，所有卡片使用同一视觉域

### 策略 B：无界分区坐标 (Borderless Zoning)

**适用场景**：强对比（生/死、对/错、旧/新）、有连贯动作转化（从 A 状态变成 B 状态）、需要物理退化效果（飘散、碎裂、溶解）的场景。卡片边框会限制这些效果的伸展。

- **画布**：默认 16:9 横向宽幅
- **划区不建墙**：不使用物理卡片。用垂直虚线（`vertical dashed line in light gray`）或水平线将画布分为 `Left Zone` / `Right Zone`（或 `Upper` / `Lower`）
- **绝对定位**：在每个 Zone 内部，用严密的方位词锁定每个物件的位置（`Upper area:`, `Center:`, `Middle:`, `Below the object:`, `Lower area: three objects in a row, evenly spaced`）
- **视觉导航**：每个 Zone 顶部加粗体标签（如 Chinese text "名词" / "动词"）
- **连接**：底部水平虚线 + 总结性文字，与策略 A 一致
- **面板内约束**：与策略 A 一致（核心物件 ≤ 3，动作暗示 ≤ 1，同一视觉域）

### 如何选择？

| 信号 | 选策略 A | 选策略 B |
|------|---------|---------|
| 原文列举了 N 个独立要点 | ✓ | |
| 原文有"A vs B"的强对比 | | ✓ |
| 需要表达物理退化（碎裂、融化、飘散） | | ✓ |
| 信息块之间有动作连贯性（从 X 变成 Y） | | ✓ |
| 拿不准 | ✓（默认） | |

## 附录 A：正交维度参考手册
以下分类不是必选菜单，是**思路卡住时的备查参考**。大多数时候，跟着原文意象和自然结构走就够了。
**结构类型**：对比/并置、流程/时间线、层级/分层、协作/交互、矩阵/象限、演变/迭代、辐射/关联、循环
**隐喻类型**：
  - **容器** — 盒子、抽屉、房间、大脑（强调存储、归类、内外边界）
  - **路径** — 道路、河流、轨道、阶梯（强调方向、进程、旅程）
  - **工具** — 望远镜、钥匙、滤网、杠杆（强调能力、聚焦、转化）
  - **生态** — 种子→树、土壤、天气、季节（强调生长、条件、自然演化）
  - **空间** — 地图、坐标系、楼层、地形（强调定位、全局视野、相对关系）

**情绪修辞**：
  - **规模感** — 极小人物 vs 巨大结构，俯瞰视角
  - **压迫感** — 密集排列、暗色调、逼近视角
  - **轻盈感** — 大面积留白、柔和色调、漂浮元素
  - **秩序感** — 精确对齐、网格布局、统一间距
  - **温度感** — 暖色调、有机形状、手绘质感

**隐喻选择优先级**：AI 渲染稳定性 > 信息传达力 > 艺术表现力
